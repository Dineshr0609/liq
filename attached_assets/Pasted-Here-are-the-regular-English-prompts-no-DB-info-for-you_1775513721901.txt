## liQ Agent Login Context Filtering - Bug Fix Summary

### Problem
The liQ Agent was returning all contracts from the database regardless of the user's active login context. When a user asked "How many active contracts do I have?", the agent would return contracts that belonged to other organizations or business units, not just their own.

For example:
- User logged in with TechSound Audio Inc. context (which has 0 contracts)
- But agent returned "4 active contracts" from other companies in the database
- The UI correctly showed 0 contracts, but the agent gave wrong data

### Root Causes

1. **Missing Organization Filter in Contract Queries** - The contract query tool was only checking for `companyId` at a basic level and not consistently applying the user's active context (company, business unit, and location) to all queries.

2. **Incomplete Context Propagation** - The agent received user context but wasn't always using it. The context helpers had early exit conditions that prevented filters from being applied when the context wasn't perfectly structured.

3. **No Fallback Logic** - If the active context wasn't available, the system didn't fall back to other available context information like the user's primary company, so it would just return all data without restrictions.

4. **Inconsistent Filtering Across Tools** - Different query tools (contracts, rules, calculations, sales data, accruals, journal entries) had inconsistent approaches to applying organization filters.
