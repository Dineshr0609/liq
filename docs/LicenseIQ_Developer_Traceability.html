TAB General Tab
Core rule identity: Rule Name (required), Description, Rule Type (percentage, tiered_pricing, rebate_tiered, minimum_guarantee, fixed_fee, payment_schedule, condition, offset, data-only, etc.), Priority (1=highest), Clause Category, Channel, Effective Date, Expiry Date, Execution Group, Base Metric (net_sales, gross_sales, units_sold, etc.).
No separate API call — fields populated from editForm state loaded on edit open. All lookups (base_metrics, clause_categories, sales_channels, execution_groups) loaded in parallel.
TAB Qualifiers Tab
Scope & applicability: Product Categories (multi-select from products table), Territories (multi-select from territory_master), Customer Segments (multi-select from customer_segments), Aggregation Period (monthly, quarterly, semi-annual, annual, per-transaction), Minimum Price ($).
No separate API call — qualifier fields are part of the editForm state. Lookups (products, territories, segments) loaded when editor opens.
TAB Conditions Tab
Boolean logic builder: Field (Volume, Amount, Territory, etc.) + Operator (Equals, >=, <=, In, Not In) + Value. For condition/data-only rule types also shows: Clause Text / Definition Text (large textarea), Severity (Informational, Standard, Important, Critical), Enforcement / Section Reference.
No separate API call — conditions stored as JSON in editForm.conditions array.
TAB Calculation Tab
Dynamic fields based on Rule Type:
• Percentage / Min Guarantee: Base Rate, Minimum Guarantee ($)
• Fixed / Annual Fee: Fee Amount, Fee Description
• Milestone Payment: Table of Event + Amount + Due Date rows
